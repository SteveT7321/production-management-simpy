# 第三章　整體設備效率：OEE（Overall Equipment Effectiveness）

## 概念說明

OEE 是衡量設備「真正創造價值時間」佔總計畫工時的百分比，由豐田生產系統（TPS）發展而來，現為 **SEMI E10** 國際標準，是製造業通用的設備健康指標。

設備的時間損失可分為三個維度：

| 維度 | 損失類型 | 常見原因 |
|------|---------|---------|
| **Availability（可用率）** | 計畫外停機 | 突發故障、等待維修、換模停機 |
| **Performance（效率）**   | 速度損失 | 速度不足、小停機、空轉 |
| **Quality（良率）**       | 品質損失 | 不良品產出、返工、報廢 |

OEE 將三者相乘，讓管理者**精確知道設備的時間在哪裡流失**，並指引改善應從哪個維度切入。

> **世界級標準：OEE ≥ 85%**（台灣 SMT 業界典型值約 40–65%）

---

## 核心公式

```
OEE = Availability（A） × Performance（P） × Quality（Q）
```

### Availability（可用率）

```
A = MTBF ÷ (MTBF + MTTR)

本產線各機台：
  高速機：A = 3,600 ÷ (3,600 + 900)  = 80.0%   ← 最低，故障頻繁
  錫膏機：A = 7,200 ÷ (7,200 + 600)  = 92.3%
  泛用機：A = 5,400 ÷ (5,400 + 720)  = 88.2%
  回焊爐：A = 28,800 ÷ (28,800 + 1,800) = 94.1%  ← 最穩定
```

### Performance（效率）

```
P = 理論最大產出 ÷ 計畫內實際產出
  ≈ 理論 Cycle Time ÷ 實際平均 Cycle Time
```

### Quality（良率）

```
Q = FPY = 良品數 ÷ 總產出數
```

### 三者相乘的複利效應

```
以高速機估算：
OEE ≈ 0.80 × 0.90 × 0.998 ≈ 71.9%

即使每個指標都看起來「還不錯」，OEE 仍可能偏低
```

## 實驗設計

**核心問題：** 提升 Availability（減少故障）vs 提升 Quality（改善品質），哪個對 OEE 改善效益更大？

- **情境 B**：模擬導入 TPM（全員預防保養）後，設備故障頻率降低
- **情境 C**：模擬導入 Six Sigma 品質改善後，不良率減半

## 結果解讀

**預期輸出：**

```
情境                      整體 OEE   vs 原始    產出率
A: 原始（基準）            ~55%        —        102 pcs/hr
B: MTBF×2（故障減半）      ~72%       ＋17%     112 pcs/hr
C: 不良率×0.5（品質↑）     ~56%       ＋1%      103 pcs/hr
```

**關鍵觀察：**
- Availability 損失（故障）遠大於 Quality 損失（不良率原本就低）
- 改善 MTBF 效益顯著，因為高速機 A 只有 80%——這是最大短板
- OEE 的複利效果：任何一項最低的維度都是改善優先目標

**各站 OEE 解讀：**
- 紅色（< 60%）：嚴重不足，需立即改善
- 黃色（60–85%）：有改善空間
- 綠色（≥ 85%）：達世界級標準

---

## 管理意涵

1. **OEE 不是整體平均，要看每站**：高 OEE 均值可能掩蓋某一站的嚴重問題
2. **三維度各有對應改善工具**：
   - Availability 低 → TPM、預防保養（見第七章）
   - Performance 低 → 標準作業、速度設定優化
   - Quality 低 → 製程管制（SPC）、防呆設計（Poka-Yoke）
3. **瓶頸設備的 OEE 最值得投資**：非瓶頸設備的 OEE 提升不影響整線產出（結合第二章 TOC）
4. **OEE 改善的 ROI**：提升 OEE 10% ≈ 等同新增一台設備的產能，但成本遠低於購機
