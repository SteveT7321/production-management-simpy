# 第七章　預防保養：PM vs 故障停機

## 概念說明

設備停機有兩種截然不同的類型：

| 類型 | 中文 | 特性 |
|------|------|------|
| **CM（Corrective Maintenance）** | 事後維修（故障停機） | 突發、不可預測、時間長、影響大 |
| **PM（Preventive Maintenance）** | 預防保養 | 計畫性、可排程、時間短、影響可控 |

傳統觀念認為「只要沒壞就不用修」，但這種「等壞才修」的策略在高稼動率的生產環境中代價極高：突發故障往往發生在最不適當的時機（高峰生產時），且修復時間遠長於計畫性保養。

**預防保養的邏輯**：在設備失效機率上升之前主動介入，用**短暫、可控的停機**換避免**長時間的突發停機**。

> 就像汽車定期換機油比等引擎壞掉再修要便宜得多。

---

## 核心公式

### 設備可用率（Availability）

```
A = MTBF ÷ (MTBF + MTTR)

本產線各機台：
  高速機：MTBF = 3,600 s（1 hr），MTTR = 900 s（15 min）
         A = 3,600 ÷ (3,600 + 900) = 80.0%
  錫膏機：MTBF = 7,200 s（2 hr），MTTR = 600 s（10 min）
         A = 7,200 ÷ (7,200 + 600) = 92.3%
  回焊爐：MTBF = 28,800 s（8 hr），MTTR = 1,800 s（30 min）
         A = 28,800 ÷ (28,800 + 1,800) = 94.1%
```

### PM 週期設定原則

```
最佳 PM 週期 ≈ MTBF × 0.7 ~ 0.8

原理：在故障率開始上升（浴缸曲線右側）之前進行保養，
     可延長 MTBF、維持高 Availability

本實驗設定：PM 週期 = MTBF × 0.8
  高速機 PM 週期 = 3,600 × 0.8 = 2,880 s（每 48 分鐘保養一次）
```

### PM 效益評估

```
PM 投資報酬（ROI）：
  減少的故障停機時間 = 故障次數減少 × MTTR
  PM 增加的停機時間 = PM 次數 × PM 時間

若 前者 > 後者，PM 有淨效益
```

## 產線實驗參數

| 情境 | PM 設定 | 說明 |
|------|---------|------|
| A | 無 PM | 純故障停機，等壞才修 |
| B | PM 週期 = MTBF × 0.8 | 在故障前主動保養 |

PM 保養時間設定：各機台 PM 時間 ≈ MTTR × 0.3（保養比修理快得多）

---

## 實驗設計

**核心問題：**
1. 導入 PM 能減少多少故障次數？
2. PM 保養停機的代價（生產時間損失）是否小於節省的故障停機時間？
3. 各站 OEE 的 Availability 維度如何變化？

## 結果解讀

**預期輸出：**

```
情境                                    故障次數   產出率     平均WIP    FPY%
A: 無 PM（純故障停機）                   ~21 次    102.4 pcs/hr  101 pcs  99.0%
B: 有 PM（PM 週期 = MTBF × 0.8）         ~9 次    109.3 pcs/hr   90 pcs  99.2%
```

**PM 建模說明：**
- PM 效果 = MTBF × 2（PM 攔截磨損，讓設備「保持年輕」，故障間隔加倍）
- 加上計畫性停機時間（PM 時間 = MTTR × 0.3，每 MTBF×0.8 做一次）

**各站 OEE 變化（A → B）：**
- 幾乎所有站點 OEE 提升（故障減少 → Availability 上升）
- 高速機（MTBF 最短）是 PM 效益最大的機台

**關鍵觀察：**
- 故障次數大幅減少 → Availability 提升 → Throughput 提升
- WIP 降低：突發故障少，PCB 不再長時間等待修復
- 計畫性 PM 停機時間 < 節省的突發故障停機時間 → 淨效益為正

## 管理意涵

1. **PM 是預防性投資，不是成本**：計算 PM 的 ROI 時，要計算「避免了多少突發停機損失」
2. **PM 週期需要按機台特性設定**：故障率高的機台（高速機）應更頻繁保養；穩定的機台（回焊爐）不需要過度保養
3. **TPM（Total Productive Maintenance）的精神**：讓操作人員也參與日常點檢，不只是維修部門的責任
4. **預知保養（Predictive Maintenance）是下一步**：
   - 傳統 PM：時間週期觸發（每 N 小時保養一次）
   - 預知保養：感測器數據觸發（振動、溫度異常時才保養）
   - 可進一步減少不必要的保養停機
5. **與 OEE 搭配使用**：PM 主要提升 OEE 的 Availability 維度（見第三章）
