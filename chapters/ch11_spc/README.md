# 第十一章　SPC 統計製程管制

## 概念說明

設備磨損、錫膏老化、環境溫濕度變化……這些因素會讓製程「悄悄漂移」：  
不良率從 0.2% 緩慢爬升到 5%，但因為每次只偏移一點，**肉眼很難察覺**。  
等到最終爆出大批不良品，往往已造成嚴重損失。

**SPC（Statistical Process Control）統計製程管制** 的做法：
- 定期從製程中取樣（n 片）
- 計算樣本統計量（如不良率 p̂、尺寸均值 X-bar）
- 與**管制界限（Control Limit）**比較
- 一旦超出界限，立即發出警報並介入

> **核心思想**：管制圖區分「普通原因變異（隨機波動）」vs「特殊原因變異（異常漂移）」。
> 只有特殊原因才需要介入；盲目調整普通原因反而讓製程更不穩定。

## 核心公式

### p 管制圖（不良率管制圖）

```
中心線（CL）：p₀ = 歷史平均不良率

上管制界限（UCL）：p₀ + 3√(p₀(1-p₀)/n)
下管制界限（LCL）：p₀ - 3√(p₀(1-p₀)/n)（通常取 max(0, LCL)）

n = 每批樣本數

本實驗：
  p₀ = 0.002（高速機正常不良率）
  n  = 20
  UCL = 0.002 + 3√(0.002×0.998/20) = 0.002 + 0.030 = 0.032
```

### 偵測能力

```
製程偏移後真實不良率 p₁ = 0.050

偵測概率（單次取樣）：
  P(p̂ > UCL | p = p₁) = P(X > UCL×n | X~Bin(n, p₁))
  = P(X > 0.032×20=0.64 | X~Bin(20, 0.050))
  ≈ 64%（約每 1.5 批可偵測到一次）

→ 期望偵測時間 ≈ 1.5 × n × 加工時間 ≈ 1.5 × 20 × 8 s ≈ 4 分鐘
```

### 製程能力指數（Cpk）

```
Cpk = min(USL-μ, μ-LSL) / (3σ)

Cpk ≥ 1.33 → 世界級（Six Sigma 要求 Cpk ≥ 1.67）
製程漂移 → μ 偏離中心 → Cpk 下降
```

## 結果解讀

**預期輸出：**

```
情境                     FPY%      產出率    WIP    SPC警報   重校停機
A: 無漂移（基準）          ~99.0%    102      101      0          0
B: 有漂移，無 SPC          ~92.0%     98      108      0          0
C: 有漂移，SPC(n=20)      ~97.5%    101      103      ~3        ~15 min
```

**關鍵觀察：**
- 情境 B：FPY 大幅下降（漂移 6 小時內大量不良品），WIP 因返工增加
- 情境 C：SPC 快速偵測漂移（~4 分鐘後發警報），重校 5 分鐘，FPY 大幅恢復
- 代價：重校停機總計約 15 分鐘，遠小於情境 B 的大量返工損失

## 管理意涵

1. **「不良率穩定」≠「製程正常」**：需要管制圖持續監控，而非事後統計

2. **n 越大，偵測越靈敏，但反應越慢**：
   - 小 n：反應快，但誤報率高（Type I Error 高）
   - 大 n：誤報少，但漂移發生後等更久才偵測到（Type II Error 高）
   - 工程上需根據不良成本 vs 誤報成本選擇適當 n

3. **管制圖只能告訴你「有問題」，不能告訴你「問題在哪」**：  
   警報觸發後需要工程師分析根本原因（Root Cause Analysis）

4. **SPC vs 100% 全檢**：
   - 全檢（AOI）：能抓到每一個不良品，但不預防問題
   - SPC：預防問題發生，從源頭控制製程，成本更低

5. **SMT 產線重點管制點**：
   - 錫膏印刷量（SPI 數據）：3D 體積管制圖
   - 貼片偏移量：X/Y 坐標 X-bar/R 圖
   - 回焊溫度曲線：峰溫、斜率管制
