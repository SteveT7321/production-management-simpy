# C03：EOQ 經濟訂購量

## 概念說明

EOQ（Economic Order Quantity）是庫存管理的基礎公式，
求解讓**訂購成本 + 持有成本最小**的最佳批量。

```
EOQ = sqrt(2 × D × S / H)

D = 年需求量（片/年）
S = 每次訂購成本（NT$）
H = 單位年持有成本（NT$/片/年）= 單位成本 × 持有率
```

總年成本：
```
TC = (D/Q) × S + (Q/2) × H
```
- `(D/Q) × S`：年訂購成本（Q 越大，次數越少，成本越低）
- `(Q/2) × H`：年持有成本（Q 越大，平均庫存越高，成本越高）

兩者的交叉點即 EOQ。

## 為什麼不用 SimPy？

EOQ 是**封閉形式公式**，直接代入計算。
敏感度分析也是靜態計算不同 Q 值的成本。

（對比：Ch10 Safety Stock 使用 SimPy 模擬需求不確定性和缺料停線）

## 預設情境（SMT 原物料採購）

| 參數 | 數值 |
|------|------|
| 年需求 D | 50,000 片/年 |
| 訂購成本 S | NT$ 800 / 次 |
| 單位成本 | NT$ 120 / 片 |
| 持有成本率 | 20% |
| 持有成本 H | NT$ 24 / 片 / 年 |
| **EOQ** | **1,826 片** |

## EOQ 曲線特性

EOQ 曲線底部**平坦**：偏離 ±20% 僅增加約 2% 成本。
實務上不必精確計算，允許供應商 MOQ（最小訂購量）靈活調整。

## 再訂購點（ROP）

```
ROP = 日需求 × 前置時間 + 安全庫存
    = (D/250) × L + SS
```

| 安全庫存策略 | SS(片) | ROP(片) |
|------------|-------|--------|
| 無安全庫存 | 0 | 1,000 |
| 0.5× LTD | 100 | 1,100 |
| 1× LTD | 200 | 1,200 |

前置時間 L = 5 天，日需求 = 200 片/天

## 與 MRP 的關係

EOQ 解決**單品項**的批量問題。
MRP（C04）解決**多品項、有相依性**的批量排程問題。
