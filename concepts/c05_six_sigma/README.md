# C05：Six Sigma / 製程能力分析

## 概念說明

Six Sigma 是以**統計方法消除製程變異**的改善方法論，
目標是 DPMO ≤ 3.4（每百萬機會不良數 ≤ 3.4）。

### 核心指標

```
Cp  = (USL - LSL) / (6σ)         製程精確度（不考慮偏移）
Cpk = min((USL-μ)/3σ, (μ-LSL)/3σ)  製程能力指數（考慮偏移）

DPMO = 規格外比例 × 1,000,000
Sigma Level：DPMO ≤ 3.4 → 6σ
```

### Cpk 判讀標準

| Cpk | DPMO | Sigma | 判定 |
|-----|------|-------|------|
| < 1.00 | > 66,807 | < 3.0 | 製程不足（需立即改善）|
| 1.00 | 66,807 | 3.0 | 勉強符合（最低標準）|
| 1.33 | 63 | 4.0 | 一般工業標準 |
| 1.67 | 0.6 | 5.0 | 汽車 / 航太標準 |
| 2.00 | 0.002 | 6.0 | Six Sigma 目標 |

## 為什麼不用 SimPy？

Cp/Cpk/DPMO 是**靜態統計計算**，輸入是製程的均值和標準差，
不涉及時間序列或隨機事件流。

（對比：Ch11 SPC 使用 SimPy 模擬製程漂移的動態偵測與控制）

## 工具功能

```
python concepts/c05_six_sigma/calculator.py
```

輸出：
1. **製程能力改善路徑**：4 個情境的 Cp/Cpk/DPMO/Sigma Level

## SMT 應用場景：錫膏印刷厚度

```
規格：LSL = 120 µm   TARGET = 150 µm   USL = 180 µm
```

| 情境 | 均值(µm) | σ(µm) | Cpk | DPMO | Sigma |
|------|---------|-------|-----|------|-------|
| 現況：製程偏移 | 158 | 12 | 0.55 | 59,540 | 3.1 |
| 改善一：置中 | 150 | 12 | 0.83 | 12,419 | 3.7 |
| 改善二：變異縮小 | 150 | 8 | 1.25 | 1,244 | 4.4 |
| 改善三：Six Sigma | 150 | 5 | 2.00 | 0.6 | 5.0+ |

## 改善優先順序

1. **製程置中**（最快、最省錢）：調整機台參數消除偏移
2. **變異縮小**（需找變異來源）：材料批次、機台磨耗、作業員差異
3. **規格放寬**（風險最高）：與客戶協商

## DMAIC 改善框架

```
Define   → 定義問題範疇（CTQ：Cpk < 1.33）
Measure  → 量測現況（收集錫膏厚度數據）
Analyze  → 分析根因（偏移來源：刮刀壓力？鋼板）
Improve  → 改善（DOE 找最佳印刷參數）
Control  → 管制（SPC 監控，見 Ch11）
```

## 與 SPC（Ch11）的關係

- **Six Sigma（本工具）**：計算現況能力、設定改善目標
- **SPC（Ch11）**：即時監控製程是否持續在管制狀態

先用 Six Sigma 工具找出改善點，再用 SPC 維持改善成果。
